# 브라우저 직접 테스트 가이드 (프론트엔드)

`doc/checkpoint_test_guide.md`가 curl/DB 위주였다면, 이 문서는 **브라우저에서 직접 클릭하며** 같은 흐름을
확인하는 버전이다. 터미널 명령은 트리거를 발동시키는 딱 한 줄만 필요하고, 나머지는 전부 화면에서 한다.

---

## 0. 사전 준비 (한 번만)

**이미 해둔 것** (그대로 두면 됨, 다시 안 해도 됨):
- `.env`에 `ANTHROPIC_API_KEY`·`ECOS_API_KEY`·`SBIZ_API_KEY`·`BIZINFO_CRTFC_KEY` 이미 채워져 있음 — 전체 플로우(리포트 생성까지) 테스트 가능한 상태
- `.claude/skills/e2e-verify/scripts/seed_demo.sql`의 시드 버그(`'CAFE'` → `'카페/디저트'`)를 **로컬에서 이미 고쳐뒀음** (커밋은 안 함 — `git diff .claude/skills/e2e-verify/scripts/seed_demo.sql`로 확인 가능). 이 픽스 없으면 아래 4번에서 트리거가 안 뜬다
- `pg-data`·`chroma-data` 볼륨을 비워둠 — 첫 기동 시 스키마가 깨끗하게 새로 올라간다

**네가 할 것**:

```bash
docker compose up -d --build      # 첫 빌드라 수 분 걸림
docker compose ps                 # postgres·chroma·ai-engine·api-core·web 5개 Up 확인
```

5개 다 `Up`이 아니면 `docker compose logs -f <서비스명>`으로 원인 확인 후 진행.

```bash
docker compose exec -T postgres psql -U bizagent -d bizagent \
  < .claude/skills/e2e-verify/scripts/seed_demo.sql
curl -X POST localhost:8000/index/rebuild
```

터미널에 `{"indexed":2}`가 찍히면 준비 끝.

## 1. 홈 화면 — 처음 상태 확인

브라우저로 **http://localhost:3000** 접속.

**보여야 하는 것**:
- 상단에 "소상공인 금융 지원" + 오른쪽에 🔔 종 아이콘
- "받은 리포트" 제목
- "아직 프로필이 없다면 온보딩 질문지부터 작성하세요" 안내문
- "도착한 리포트가 없습니다..." 문구 (아직 리포트가 없으니 정상)
- 🔔 아이콘에 빨간 배지 숫자 **없음** (정상 — 아직 알림이 없다)

---

## 2. 온보딩 질문지 — 실제로 채워서 제출해보기

1. 홈 화면의 **"온보딩 질문지"** 링크 클릭 → `/onboarding`으로 이동
2. Q1~Q9를 직접 채운다:
   - Q1 업종: 드롭다운에서 아무거나 (예: 카페/디저트)
   - Q2~Q5: 각각 드롭다운 선택
   - Q6 소재지: 텍스트 입력 2칸 — 예: `서울` / `마포구` (자유 텍스트라 아무거나 입력해도 저장됨, 검증 없음 — 알려진 갭)
   - Q7 경영 고민: 체크박스, **최대 2개까지만 선택됨**(3개째부터는 가장 오래된 게 자동으로 빠짐 — 직접 확인해볼 것)
   - Q8 정책자금 신청 경험: 드롭다운 선택
   - Q9 사업자등록번호: 숫자 10자리 (형식만 체크, 실제 국세청 조회는 안 함 — 아무 숫자나 돼도 저장됨)
3. **"등록"** 버튼 클릭

**기대**: 에러 문구 없이 홈(`/`)으로 자동 이동하면 성공.

**⚠️ 중요**: 이렇게 방금 만든 프로필은 **트리거 데모에 못 쓴다.** 상권 코드(`market_region_code`)가
자동으로 안 채워지기 때문이다 (S3 미완 — `checkpoint_test_guide.md` 4번 참고). 아래 3~5번은 전부
**시드로 미리 만들어둔 프로필(1번)**로 진행하고, 방금 만든 프로필은 "온보딩 화면 자체가 동작하는지"만
확인하는 용도다.

## 3. 트리거 발동시키기 (터미널 한 줄)

화면에 "지금 발동" 버튼이 없어서, 이 한 단계만 터미널이 필요하다 (원래 매일 06:00 자동 실행되는 걸
즉시 실행하는 것뿐):

```bash
curl -X POST localhost:8080/api/agent/check/1
```

Claude를 두 번(원인분석 L3 + 리포트생성 L5) 호출해서 **10~30초 정도 걸릴 수 있다.** 응답이 아래처럼
오면 성공:

```json
[
  {"event":"new_competitors_500m:GTE:3.0","status":"PROCESSED","reportId":1},
  {"event":"foot_traffic_delta_pct:LTE:-15.0","status":"PROCESSED","reportId":2}
]
```

## 4. 알림 벨에서 실시간으로 확인하기

브라우저로 돌아가서 **새로고침**(또는 최대 30초 대기 — 벨이 30초마다 자동 폴링한다):

- 🔔 아이콘 오른쪽 위에 **빨간 배지 숫자**(2)가 뜨는지
- 화면 우측 하단에 **토스트 팝업**("🔔 ...")이 잠깐 떴다가 사라지는지 (새로고침 없이 30초 폴링 타이밍에
  맞으면 자동으로 뜬다 — 못 봤으면 새로고침해서 배지만 확인해도 무방)
- 🔔 아이콘 **클릭** → 알림 목록 드롭다운이 열리는지, 알림 2개(제목+본문 미리보기)가 보이는지
- 알림 항목 **클릭** → 자동으로 `/reports/{id}` 페이지로 이동하는지, 그리고 방금 클릭한 알림이
  목록에서 사라지는지(읽음 처리)

## 5. 리포트 화면 확인

4번에서 이동한 리포트 페이지(또는 홈에서 "리포트 #1" 링크 클릭)에서 확인할 것:

- 리포트 본문이 마크다운 서식(제목·굵게·목록)으로 렌더링되는지 (날것의 `##`, `**` 기호가 그대로
  보이면 렌더러 버그)
- 원인 설명이 "사장님이 이해할 수 있는 말투"로 읽히는지 (PRD 성공기준 ③)
- 하단 "매칭된 정책자금" 섹션에 공고 카드가 있는지, 각 카드에:
  - 제목(클릭하면 외부 링크로 이동하는지)
  - "신청 마감: YYYY-MM-DD"
  - "근거: ..." (RAG가 왜 이 공고를 골랐는지 설명하는 문장)
  가 다 채워져 있는지 — 하나라도 비어 있으면 위쪽 항목 순서대로 어디가 비었는지가 원인(마감일 없으면
  수집 데이터 문제, 근거 없으면 RAG evidence 생성 문제)

## 6. (선택) 중복 알림 방지 확인

3번의 curl을 **한 번 더** 실행하고 브라우저를 새로고침해봐도 배지 숫자가 늘지 않으면 정상
(14일 내 같은 트리거 재알림 차단). 응답 JSON에도 `"status":"DUPLICATE_SKIPPED"`가 찍힌다.

---

## 7. (선택) 개발자도구로 "규칙 위반 없는지" 확인

브라우저 개발자도구(F12) → Network 탭 열어놓고 1~6번을 다시 해보면서, 요청이 전부
`localhost:8080`(Spring)으로만 나가는지 확인. `localhost:8000`(ai-engine)으로 직접 나가는 요청이
있으면 안 된다 — 이 프로젝트 규칙상 웹은 반드시 Spring만 거쳐야 한다.

## 문제 생겼을 때

| 증상 | 원인/조치 |
| --- | --- |
| 3번 curl 응답이 `[]` | 시드가 잘못 들어갔거나(위 0번 다시 확인) 이미 발동된 트리거임(`docker compose exec postgres psql -U bizagent -d bizagent -c "DELETE FROM trigger_event WHERE profile_id=1;"`로 초기화 후 재시도) |
| 3번 curl이 500 | `.env`의 `ANTHROPIC_API_KEY`가 비었거나 잘못됨. `docker compose logs ai-engine`으로 원인 확인 |
| 알림 벨에 배지가 안 뜸 | 30초 폴링 타이밍 문제일 수 있음 — 새로고침해서 다시 확인. 그래도 안 뜨면 F12 Network 탭에서 `/api/notifications` 응답 직접 확인 |
| 리포트 매칭 공고가 0건 | `/index/rebuild`를 안 돌렸거나 chroma-data가 예전 384-dim 컬렉션을 물고 있음 → `docker compose down && rm -rf chroma-data` 후 재기동 |
| 화면이 아예 안 뜸 | `docker compose logs web` — api-core 기동 대기 중일 수 있음 |
